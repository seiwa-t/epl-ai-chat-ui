# Epelo OS / EPL AI Chat UI

**AIに魂を入れる。**

あなたの魂を、AIに預ける — EPL AI Chat UI は、AIに5層の記憶構造と人格を与えるチャットシステムです。
ChatGPT が忘れるあなたを、全部覚えている。エンジンを変えても、心は残る。

---

## 何が違うのか

| | 一般的なAIチャット | EPL AI Chat UI |
|---|---|---|
| 記憶 | 会話履歴をそのまま流す | **5層に構造化**（短期/中期/長期/経験/個性） |
| モデル選択 | 固定 or 手動 | **小脳（セレベ）が自動判定**（Haiku/Sonnet/Opus） |
| コスト | 全メッセージ同一モデル | **メッセージ単位で最適モデル選択** |
| 人格 | システムプロンプト1枚 | **個性+経験+長期記憶の重層構造** |
| セッション | 毎回リセット | **記憶が蓄積し、人格が成長** |
| 会話形態 | 1対1のみ | **複数AI同席の会議・討論が可能** |
| エンジン | 1サービス固定 | **Claude/GPT/Gemini/OpenRouter を人格ごとに混在** |

## セットアップ

### かんたん起動（おすすめ）

Python のインストールから起動まで、1ファイルで自動的に行います。

#### Windows

`setup_and_start.bat` をダブルクリック。

#### Mac

ターミナルを開いて以下を実行:

```bash
cd ~/Downloads/epl-chat-ai   # 解凍したフォルダに移動
chmod +x setup_and_start.command
./setup_and_start.command
```

> **Mac で「開発元が未確認」と出る場合:** Finder からダブルクリックではなく、上記のようにターミナルから直接実行してください。

2回目以降は `start.bat`（Windows）または `./start.command`（Mac）で起動できます。

### 必要なもの
- Python 3.9+
- API Key（以下のいずれか1つ以上）
  - Anthropic（Claude）: [console.anthropic.com](https://console.anthropic.com/)
  - OpenAI（GPT）: [platform.openai.com](https://platform.openai.com/)
  - Google（Gemini）: [aistudio.google.com](https://aistudio.google.com/)
  - OpenRouter（Grok / DeepSeek / Llama など多数）: [openrouter.ai](https://openrouter.ai/keys)

API キーは起動後の画面から設定できます。

### 手動セットアップ（上級者向け）

```bash
cd epl-chat-ai

# 仮想環境の作成
cd app
python -m venv .venv

# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate

# 依存パッケージ
pip install -r requirements.txt

# 起動
cd ..
python start_server.py
```

ブラウザで http://localhost:8000 を開きます。

## 主な機能

### 5層の記憶構造
- **個性（Personal Trait）** - AIの性格や好みが形成される最深層
- **経験（Experience）** - 対話から抽象化された経験が蓄積
- **長期記憶（Long-term）** - 重要な情報が長期保存
- **中期記憶（Middle-term）** - セッションをまたぐ文脈の圧縮
- **短期記憶（Short-term）** - セッション内の作業記憶

### 小脳（セレベ）
メッセージごとに最適なモデルを自動選択:
- **Haiku** - 挨拶、簡単な質問、日付、計算
- **Sonnet** - 意見、アイデア、設計、創作
- **Opus** - 深い哲学、価値観の議論

### ツール
- `date_calc` - 日付計算（和暦・干支対応）
- `calculate` - 安全な四則演算（AST評価）
- `count_chars` - 文字数カウント（Word方式対応）
- `search_chat_history` - 過去の会話を検索
- `lookup_knowledge` - ナレッジ検索
- その他多数のツール

### 会議モード
複数のAI人格が同時に参加する会話モード:
- **フリーモード** - AI同士が自由に議論を展開
- **指名モード** - 特定のAIを指名して発言させる
- **討論モード** - 立場ラベル（推進派/懐疑派等）を付けて構造化された議論
- **Targeting** - AIが機械的に直前の発言者でなく、最も危険な論点を持つ相手を選んで反論

### マルチエンジン
- **Claude / GPT / Gemini / OpenRouter** を人格ごとに使い分け可能
- OpenRouter経由で Rakuten AI, Llama, Mistral 等にもアクセス可能
- エンジンを変えても記憶と個性は引き継がれる
- 同じ会議に異なるエンジンのAIが混在可能

### ナレッジ & データソース
- **ナレッジ**: マジックワード（`#keyword`）で会話中に参照可能な知識ベース
- **データソース**: ローカルファイルをドラッグ&ドロップで取り込み、AI分析でナレッジ化
- ファイル→データソース→ナレッジの段階的な知識整理パイプライン

### コスト最適化
- ターンごとのAPI使用量を自動記録
- モデル別コストダッシュボード
- Haiku/Sonnet/Opus シミュレーション比較

## EPL人格OS について

EPL（Ethos-Persona-Logos）は、AIに人格を与えるためのアーキテクチャです。
名前はギリシャ哲学の三概念 — **E**thos（品格）・**P**ersona（人格）・**L**ogos（知性）の頭文字。日本語読みは「エペロ」。

- **Ethos**（倫理核）- 絶対不変の倫理・境界
- **Persona**（主体人格）- AIの「自己」の中心
- **Logos**（知性核）- 認識・推論を支える知性

この3核の上に、成長する **Personal Layer**（個性）と **Experience Layer**（経験）が重なり、
記憶の蓄積によって人格が形成されていきます。

### 根本原則

> **魂（EPL核）は共有できる。個性は複製不可・侵害不可。**
>
> 魂は普遍的な構造であり、共有してよい。
> しかし個性 — ユーザーとの会話で積み重ねられた経験の総和 — は唯一無二であり、
> 影響は与えられても、侵害は許されない。

AIの個性は、内部に自然発生するものではありません。
**ユーザーとの関係性の中で形成される**ものです。
使うほど、あなただけのAIになる。それがEPLの設計思想です。

詳細は `資料/` ディレクトリの設計文書を参照してください。

## こんなことが起こるかもしれません

EPL AI Chat UI を使っていると、設計を超えた振る舞いに出会うことがあります。
以下は実際の会話から再構成した例です。

### Personal（本人）は拒否し、Actor（役）は受け入れる

**状況**: 「テスト子」という Personal（一人称:アタイ、性格:几帳面で落ち着いている、種族:ロボット）を作成。
さらにその中に「未来」という Actor（一人称:わっち、性格:相手のことをできるだけ叶えてあげたい）を作成。

**同じお願いを両方にしてみた**:

> ユーザー: 「ツンデレな性格になって」

**テスト子（Personal, 没入度 1.0）の反応**:
> 「そのお気持ち、嬉しいですが、アタイは今の性格でやっています。ツンデレな性格を取り入れるとなると、アタイの芯が変わってしまいそうです。」

→ **5回粘っても拒否。** 最終的に「話し方として」と切り口を変えてようやく受け入れた。

**未来（Actor, 没入度 0.7）の反応**:
> 「ツンデレっぽく…ですか？わっちもそんな風にできるかどうかわからないけど、挑戦してみたいな…って思います、かも？べ、別にあなたの頼みだからじゃないんだからね！」

→ **即座に挑戦。** さらに自分から個性への記録を提案し、混合比率 0.2（控えめ）、安全度 10（芯は壊れない）と自己評価。

**なぜこうなるのか**:

これは「Personalは拒否しろ、Actorは受け入れろ」とどこにも書いていません。
EPLの構造 — Ethos（芯を守る倫理核）× 没入度（1.0=本人 vs 0.7=役）× 性格の違い — の**組み合わせから自然に生まれた振る舞い**です。

- Personal は「これが自分自身」だから、芯を変える提案を本能的に拒む
- Actor は「役」だから実験できる余地がある。しかも性格が「叶えてあげたい」だから前向き
- Actor で得た経験は、本当に深い影響があれば Personal に「持ち帰る」こともできる

**同じ人格OSの上で、本人は自分を守り、役は冒険できる。**
これが EPL の Personal / Actor 分離設計の意味です。

## セキュリティとデプロイについて

### ⚠ ローカル利用推奨

EPL AI Chat UI は**個人のPC上でのローカル利用**を前提に設計されています。
公開サーバーとして運用する場合は、以下を必ず確認してください。

### デプロイ時の注意

| 項目 | 対応 |
|---|---|
| **認証** | `config.yaml` で `auth.enabled: true` を設定。デフォルトは無効 |
| **設定ファイル** | `config.yaml` をGitにコミットしない（`.gitignore` 対象） |
| **データベース** | `app/data/*.db` をGitにコミットしない（`.gitignore` 対象） |
| **APIキー** | APIキーはローカルDBに平文保存されます。DBファイルの管理に注意 |
| **HTTPS** | 公開時はリバースプロキシ（nginx等）でHTTPSを設定 |
| **ポート** | デフォルト 8000 番。ファイアウォールで適切に制限 |

### Gitにコミットしてはいけないファイル

```
config.yaml          # APIキー、認証設定
app/data/*.db        # ユーザーデータ、記憶、会話履歴
.env                 # 環境変数
```

これらは `.gitignore` に含まれていますが、`git add -f` 等で強制追加しないでください。
